# 2_preprocessing

영상 및 이벤트 데이터를 모델 학습용 데이터로 변환하는 전처리 폴더입니다.

---

## 1. 역할

이 폴더에서는 원본 영상 또는 프레임 데이터를 바탕으로
모델 학습과 예측에 필요한 데이터를 생성합니다.

주요 기능은 다음과 같습니다.

- 프레임 정리
- 이벤트 CSV 기반 라벨 생성
- MediaPipe 손 랜드마크 NPZ추출
- 텐서 데이터 생성
- XML → YOLO txt 변환

---

## 2. 주요 입력 데이터

- `video/` 폴더의 원본 영상 또는 프레임
- 이벤트 플래그 CSV
- XML 어노테이션 파일
- 기타 전처리 대상 원본 데이터

---

## 3. 주요 출력 데이터

- 랜드마크 NPZ
- 라벨 CSV
- 텐서 데이터
- YOLO 형식 txt 라벨
- 전처리 중간 결과 파일

---

## 4. 전처리 흐름

### 1) 프레임 데이터 준비
영상 또는 촬영 데이터를 프레임 단위로 정리합니다.

### 2) 이벤트 라벨 정리
이벤트 CSV를 이용해 프레임 또는 구간 단위 라벨을 생성합니다.

### 3) 손 랜드마크 추출
MediaPipe를 이용해 손 위치 및 움직임 정보를 추출합니다.

### 4) 시계열 입력 생성
랜드마크 또는 이미지 기반으로 시계열 모델 입력 텐서를 생성합니다.

### 5) YOLO 학습용 라벨 생성
어노테이션 파일을 객체 탐지 학습 형식으로 변환합니다.

---

- CSV 주요 컬럼 설명
- NPZ 내부 키 구조
- 윈도우 길이, stride, fps 등 주요 파라미터
- 누락 프레임 처리 방식

---

## 6. 예시 설명

예시 데이터 흐름:

- 원본 프레임 → 손 랜드마크 추출 → NPZ 저장
- 이벤트 CSV → 프레임 단위 라벨 생성
- 프레임 + 라벨 → 텐서 데이터 생성
- XML → YOLO txt

## 7. 주의사항

- 파일명 오타는 정리하는 것이 좋습니다.
- 중복 함수가 존재하면 공통 모듈로 분리하는 것이 좋습니다.
- 생성 파일이 `data/`에 저장되는 경우 파일명 규칙을 README에 반드시 적어야 합니다.
- Tensor 전용 폴더가 없다면 어떤 산출물이 `data/` 아래에 저장되는지 명확히 설명해야 합니다.
